\begin{abstract}
This paper presents a reproducible end-to-end framework for tourism review analysis using structured and unstructured user-generated content. The workflow includes dataset preparation, location and temporal feature engineering, sentiment-oriented labeling, statistical analysis, and model validation. We process review text and metadata to construct robust predictors such as review delay, review length, user contribution behavior, and location-derived signals. Experimental results on a real-world tourism review dataset show that the proposed feature set improves predictive performance and supports actionable insights for destination management. The framework is designed for reproducibility and can be adapted to other regional tourism datasets with minimal changes.
\end{abstract}

\section{Introduction}
Tourism platforms generate large volumes of user reviews that contain valuable behavioral and sentiment signals. Transforming these raw signals into reliable analytical outputs requires careful data preparation, domain-aware feature engineering, and rigorous validation.

This study develops a practical framework for tourism review analytics with emphasis on reproducibility and conference-grade reporting. The framework is demonstrated on Sri Lankan tourism review data and supports both statistical interpretation and predictive modeling.

\textbf{Contributions of this paper are as follows:}
\begin{itemize}
    \item A reproducible preprocessing pipeline for tourism reviews with deterministic handling of location and country extraction.
    \item A comprehensive feature engineering strategy combining location, temporal, and text-derived indicators.
    \item An empirical evaluation protocol for comparative model validation and interpretation.
\end{itemize}

\section{Related Work}
Prior work in tourism analytics has used sentiment mining, aspect extraction, and predictive models on online reviews~\cite{liu2012sentiment,cambria2017sentic}. Recent studies emphasize combining metadata with textual features to improve model robustness~\cite{zhang2021review}. However, many pipelines are difficult to reproduce due to inconsistent preprocessing and weak reporting of feature construction.

Our work addresses this gap through a transparent feature engineering workflow and explicit export of processed datasets for downstream tasks.

\subsection{Dataset and Preprocessing}
The dataset consists of tourism reviews featuring location names, user locales, ratings, text, and timestamps. Records are initially subjected to missing value imputation and duplicate removal. Date parsing extracts the travel year and month, publication dates, and calculates the temporal review delay in days. Additionally, travel seasons (e.g., peak dry, inter-monsoon, southwest monsoon) are derived from the travel month. Location normalization maps raw strings to consistent province, district, and generalized tourism regions. User country and region inference utilize dictionary-based mappings combined with string matching rules via the \textit{pycountry} library.

\subsection{Feature Engineering}
A comprehensive set of predictors is generated to capture multiple facets of the visitor experience:
\begin{itemize}
    \item \textbf{Behavioral and Text Features:} Review text and title word counts, discretized length buckets, user contribution levels, and a normalized helpfulness ratio. A composite review quality score is also computed based on text richness and helpful votes.
    \item \textbf{Destination Performance Features:} Destinations are aggregated to calculate average ratings, total review counts, and ranking metrics (rating vs. popularity gap).
    \item \textbf{Temporal Features:} Travel season, review delay in days, and longitudinal patterns.
\end{itemize}

\subsection{Natural Language Processing and Topic Modeling}
To derive robust textual insights, an ensemble sentiment classification approach is employed using three pre-trained transformer pipelines: \textit{CardiffNLP Twitter-RoBERTa}, \textit{SieBERT}, and \textit{BERTweet}. A mean voting mechanism creates a consolidated sentiment label and score, protecting against model biases. Emotion classification is executed using a pre-trained DistilRoBERTa model. These outputs enable the calculation of the sentiment-rating gap to identify incongruence.

Furthermore, latent document topics are extracted using \textbf{BERTopic} equipped with the \textit{all-MiniLM-L6-v2} sentence transformer. Discovered topic keywords are procedurally mapped to broad review themes, such as hospitality, wildlife, or heritage.

\subsection{Statistical Analysis, Modeling and Validation}
The final phase interprets and predicts review attributes. To explore relationships such as rating-sentiment incongruence across venue categories, categorical statistical tools like the Chi-square test of independence and odds-ratios are used. For predictive tasks, a supervised learning setup is utilized. Data is split into training and testing partitions using cross-validation. Performance is systematically evaluated using Accuracy, Precision, Recall, and F1-score metrics.

\subsection{Implementation Details}
Experiments are implemented in Python with common data science libraries. The preprocessing and feature generation pipeline is executed before model training to avoid leakage.

\subsection{Evaluation Metrics}
For classification, we report:
\begin{equation}
\text{Accuracy} = \frac{TP + TN}{TP + TN + FP + FN}
\end{equation}
where $TP$, $TN$, $FP$, and $FN$ denote true positives, true negatives, false positives, and false negatives.

\section{Results and Discussion}

\subsection{RQ2: Does incongruence vary significantly across location types?}

To examine whether sentiment--rating incongruence differs across venue categories, we conducted a chi-square test of independence between \textit{Location Type} and the binary incongruence label. The association was statistically significant, $\chi^2(10)=125.85$, $p<0.001$, indicating that the probability of observing incongruent reviews is not evenly distributed across attraction types. However, the effect size was small (Cram\'er's $V=0.088$), suggesting that location type explains only a modest proportion of the overall variation in incongruence.

Table~\ref{tab:rq2_location_type} summarizes the incongruence rates across venue categories. National Parks recorded the lowest incongruence rate (12.8\%) and were therefore used as the reference category for odds-ratio comparison. In contrast, Museums showed the highest rate (26.3\%), followed by Bodies of Water (23.7\%) and Zoological Gardens (21.6\%). Relative to National Parks, these categories showed substantially higher odds of sentiment--rating mismatch, with odds ratios of 2.43, 2.12, and 1.88, respectively.

Religious Sites (19.8\%), Beaches (19.0\%), and Waterfalls (18.0\%) formed an intermediate group with above-average incongruence levels. Lower-risk categories included Gardens (17.4\%), Farms (16.6\%), Nature \& Wildlife Areas (16.4\%), and Historic Sites (15.6\%), though all remained above the National Parks baseline. Overall, the findings indicate that incongruence is systematically concentrated in particular tourism settings, even if the overall strength of association remains limited.

A plausible interpretation is that certain attraction types encourage more emotionally mixed or narratively complex reviews, causing the written sentiment to diverge from the assigned rating. Museums, waterfront attractions, and zoos may evoke nuanced visitor experiences that are not fully captured by a single numeric score. By contrast, National Parks appear to generate more consistent alignment between narrative sentiment and explicit rating behavior.

The practical implication of this result is that location type should be treated as a meaningful contextual predictor in downstream modeling and interpretation. Although the effect size is not large, the clear differences across venue categories suggest that destination-specific context contributes to how visitors translate their experiences into text and ratings.

\section{Threats to Validity}
Potential threats include dataset representativeness, annotation assumptions in rating-class conversion, and location parsing noise in ambiguous records. External validity should be assessed by testing the framework on additional tourism datasets.

\section{Conclusion and Future Work}
This paper introduced a reproducible framework for tourism review analytics integrating preprocessing, feature engineering, and model validation. Future work includes multilingual sentiment modeling, aspect-level opinion mining, and temporal drift analysis.
